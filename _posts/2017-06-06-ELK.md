## 避坑

1. elasticsearch和kibana的版本更新是保持一致的，所以安装的时候保持版本一致，不然会有一些警告出现
2. 5.4.1版本要求Linux内核大于3.5
3. 文件描述符要求大于 65536
4. vm.max_map_count要求大于262144， `vi /etc/sysctl.conf ，添加vm.max_map_count=655360`
5. java环境 1.8以上



## 安装配置

组件  | 主机 | 功能
---|--- | ---
elasticsearch | 10.90.1.199 | 存储和索引
kibana | 10.90.1.199 | 图形展示
redis(broker) | 10.90.1.199 | BROKER 
logstash-indexer| 10.90.1.199| Indexer
logstash-shipper| 10.90.1.146 | 被采集端, n个


#### 数据流
 
logstash-shipper ==> broker(redis) ==> logstash-indexer ==> elasticsearch


####　logstash-indexer 配置

相当于server端的一个配置

```
input {  
    redis {
        host => "10.90.1.199"
        port => 6379
        data_type => "pattern_channel"
        key => "logstash-*"
    }
}  

output{  
    elasticsearch {  
         hosts => ["10.90.1.199:9200"]  
         index => "logstash-chan"  
    }  
    stdout {  
        codec => rubydebug  
    }  
} 
```


#### logstash-shipper 配置

相当于agent端的一个配置


```
input {  
    file {
        path => "/var/log/messages"
    }
}  

output{  
    redis {
	host => "10.90.1.199"
        port => 6379	
        data_type => "channel"
        key => "logstash-chan-%{+yyyy.MM.dd}"
    }
    stdout {  
        codec => rubydebug  
    }  
} 
```

这里有个问题就是，在logstash-shipper的key配置为`logstash-chan-%{+yyyy.MM.dd}`， data_type为`channel`后，
logstash-indexer也为相同配置的时候，就无法识别。从而导致无法消费redis中的数据，所以这里配置为data_type为`pattern_channel`,  
key为`logstash-*`进行模糊匹配。
